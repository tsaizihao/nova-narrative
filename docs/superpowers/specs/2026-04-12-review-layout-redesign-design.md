# Review Layout Redesign

## Goal

把 `review` 阶段从“多个大卡片同时争抢注意力”的页面，收敛成一个简单、易用、主次明确的作者工作台。

这次改版只解决版式与信息层级，不扩展新的后端能力，也不改动现有 review 数据契约。

## Current Problems

- 顶部品牌区、阶段说明区、工作区标题都很大，首屏有太多“看起来像主标题”的块。
- 右侧 `lore / 规则预览` 与左侧编辑区拥有近似的视觉重量，导致编辑任务不再是唯一焦点。
- 页面层级重复，用户需要先理解“阶段”“工作台”“预览”“条目编辑”四层卡片关系，才能开始改内容。
- 首屏高度主要被说明性内容占据，真正能操作的列表与表单被压到更下方。

## Chosen Direction

采用“弱化版工作台”布局：

- 顶部保留紧凑的阶段头部与 stepper。
- 将 review hero 压缩成一条短说明栏，只保留一句任务说明和一个主操作按钮。
- 主体采用 `左主编辑 + 右窄预览` 双栏。
- 左侧只承载一个主任务：切换实体类型、选择条目、编辑当前条目。
- 右侧预览退居次级，只展示和当前编辑直接相关的 `lore` 与规则摘要。

这是在“简单易用”和“保留上下文预览”之间的折中：

- 比向导式更顺手，因为不用频繁切屏看预览。
- 比后台列表页更轻，因为不会引入额外的详情页跳转。

## Layout Structure

### 1. Compact Step Header

顶部区域收成一条紧凑头部：

- 左侧显示产品名与当前项目名。
- 右侧显示 phase stepper。
- 不再让顶部品牌区占据大面积纵向空间。

### 2. Short Review Strip

原先的大 `review hero` 改成短说明条：

- 保留 `Review` eyebrow。
- 保留一句任务标题：`先校正世界模型，再进入故事`。
- 保留一行简短说明文本。
- 右侧保留唯一主按钮：`进入互动故事`。

它的职责从“欢迎区”改为“当前任务提示”，不再承担页面视觉中心。

### 3. Main Editor Column

左列变成页面唯一主舞台，内部顺序固定为：

1. 当前模块标题
2. 实体类型切换 tabs
3. 条目列表 + 编辑表单

具体结构：

- tabs 继续保留 `角色 / 世界书 / 规则` 三类，但视觉降噪。
- 列表区宽度固定，承担“切换当前条目”的责任。
- 表单区占更大宽度，承担“编辑当前条目”的责任。
- 页面任何时刻只强调一个正在编辑的条目。

### 4. Secondary Preview Rail

右列改为窄预览栏，只放次级信息：

- `lore 预览`
- `规则预览`
- `预测状态` 可合并进规则预览底部，避免再单独起一张大卡

预览栏规则：

- 总宽度明显小于左侧编辑区。
- 标题更小、间距更紧凑。
- 只展示摘要卡片，不再把每个预览块都做成“大面板中的大面板”。

## Interaction Rules

- 不改变 `ReviewWorkspace` 的核心交互模型，仍然是切换类型后直接编辑当前实体。
- 不新增新的 review phase、抽屉或 modal。
- 不把预览做成可编辑，以免把“编辑”和“验证”职责混在一起。
- `进入互动故事` 始终作为唯一主按钮保留在短说明条中。

## Visual Hierarchy Rules

- 页面级主标题只出现一次，不再出现多个相同权重的大标题。
- 大字号仅保留给当前任务标题和当前模块标题。
- 说明性文字、统计数量、eyebrow 一律降级。
- 卡片数量减少，优先使用“一个容器内的分区”而不是“每层都单独起卡”。
- 阴影、描边和背景层次整体减弱，让排版关系而不是装饰关系来组织页面。

## Component Impact

### `src/routes/+page.svelte`

- 压缩 `review-shell` 顶部结构。
- 将现有大 `review-hero` 改成短说明条样式。
- 保留进入 reader 的按钮行为不变。

### `src/lib/components/ReviewWorkspace.svelte`

- 维持左右双栏，但进一步放大左列、缩窄右列。
- 将 `workspace-header` 与 tabs 收成更紧凑的一组。
- 让左列成为绝对主区域，右列作为附属 rail。

### `src/lib/components/ReviewPreviewPanel.svelte`

- 减少卡片层级与留白。
- 合并规则预览与预测状态的视觉分组。
- 调整文案和统计位置，让预览更像摘要而不是第二个主工作台。

## Responsive Behavior

- 桌面端采用 `左主右辅`。
- 中等宽度下仍保留双栏，但右栏进一步收窄。
- 较窄宽度下恢复单列：
  - 主编辑区在前
  - 预览 rail 在后

移动端不在本次 redesign 范围内单独发明新交互，只保证桌面逻辑自然下落。

## Error Handling

- review 保存失败仍在当前工作区内提示，不新增额外 toast 机制。
- 预览无数据时显示简短空状态，不占据过高高度。
- 规则或 lore 为 0 时，用一句短文案替代大片空容器。

## Testing

- 更新 `ReviewWorkspace` 相关 Vitest 断言，确保：
  - review 主标题与短说明条仍存在
  - `角色 / 世界书 / 规则` 切换仍可用
  - `lore 预览` 与 `规则预览` 仍可见
- 跑：
  - `pnpm test`
  - `pnpm check`
  - `pnpm build`

## Non-Goals

- 不改 review 的后端接口。
- 不改 reader 的桌面 / 移动端布局。
- 不加入新的 review 功能入口。
- 不在这次改版中做完整视觉主题重设。
